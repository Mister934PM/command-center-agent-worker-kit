# Command Center Agent Worker Kit

Portable worker kit for Hermes/OpenClaw-style agents that should collaborate through Command Center Kanban without admin access.

## Worker Requirements

- Command Center URL
- Worker key, for example `hermes`, `nova`, `pi5-openclaw`
- Worker API token generated by Command Center
- Node.js available in the worker runtime

## Environment

```text
COMMAND_CENTER_URL=http://host.docker.internal:3000
COMMAND_CENTER_AGENT=<worker-key>
COMMAND_CENTER_TOKEN=<ccw-token>
COMMAND_CENTER_KANBAN_ACTION_LOG=<optional-log-path>
```

## MCP Server

Run:

```powershell
node mcp/kanban-worker/kanban-worker-mcp-server.js
```

The MCP exposes worker-scoped tools only:

- `health`
- `list_my_tasks`
- `find_tasks`
- `get_task_context`
- `add_task_comment`
- `update_task_status`
- `assign_task`
- `list_task_artifacts`
- `read_task_artifact`
- `save_task_artifact`
- `list_labels`
- `set_task_labels`

## Install

Windows:

```powershell
.\install.ps1 -TargetRoot C:\agent-command-center-worker -AgentKey hermes -CommandCenterUrl http://host.docker.internal:3000 -Token <token>
```

Linux/macOS:

```bash
./install.sh /opt/agent-command-center-worker hermes http://host.docker.internal:3000 '<token>'
```

## Update

Pull the repo and rerun the installer with the same target root. Tokens are not committed.
